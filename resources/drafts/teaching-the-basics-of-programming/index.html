Teaching the Basics of Programming:
My Current Approach

Written on Thu 30 April 2020

Part

education opinion

In the first part of this series we looked at the reasons for using Python to teach programming. In this part we're going to look at the material to be covered in more detail. Furthermore I'll try to demonstrate my approach along with my reasoning for why I believe this approach is far superior.

Introduction

In my experience, introductions to programming courses often follow an unhelpful structure. Far too many, at least in my opinion, focus on the grammar and syntax of a programming language. I think this is akin to teaching students how to write stories by getting them to read a dictionary. In my experience, of the students who do actually try, this approach results in gaps in their knowledge due to failure to apply newly learnt techniques.

Syllabus

Before we delve into the course structure, what exactly do we want students to get out of a basic introduction to programming? This is my current syllabus for teaching year 10 / 11 (aged 14 to 16 years old) students an introduction to Python.

  1. Variables
    • What a variable is
    • Using the print and input functions
    • Case and spelling matter! Name, NAME and name are not the same variable.
    • Assignment of values is NAME = VALUE not anything else! Content on right is assigned to the variable on the left.
    • Valid and invalid variable names. No spaces, cannot start with a number. Good practice when naming variables.
    • Swapping contents using a,b = b,a
    • [Opinion] Students should use long, descriptive variable names. All in lowercase joined by underscores. Students should learn to "tab complete" variables, not type them out.
    • Understand how inequality operators work ( ≤ ,  ≥ ,  < ,  > )
    • Understand the augmented assignment operators work ( + \! = ,  − \! = ,  ⁄ \! = ,  ⁄  ⁄ \! = , *\! = )
    • How to typecast variables from one type to another. Such as int to float.
    • Order of operation matters. Why you cannot access a variable before it is created!
  2. Conditionals
    • How to write tests to produce outcomes that are True or False (a  >  0, b in c, d !\! =  e, ⋯)
    • Understand the difference between the = and == operators.
    • Understand the difference between the != and not operators
    • Syntax using tests, colons and indentation.
    • When to use an if / else statement
    • What is the point of the elif statement?
    • The importance of the tabs
    • Using if statements to catch edge cases.
    • Nested If Statements as a concept (to be able to follow program logic by hand)
    • Using and, or
  3. While loops
    • What is a loop, what is a while loop and what are they for
    • Determining the difference between a while loop or an if statement in a flowchart
    • Ensuring the test is True for as long as we need to repeat the code
    • How to create (and escape from!) an infinite loop
    • When to use a do-while vs a while loop
    • Nested loops
  4. Lists
    • What a list is
    • Why we group items together in lists
    • How to 0-index items in a list. How Python uses negative indices.
    • How to select a single or multiple items from a list
    • Using first, *middle, end = data to split a list into parts
    • Some commons functions to apply to lists (sum, min, max, len, ...)
    • How to add and remove items to a list
    • Testing if a value is inside a list
    • How to sort a list
  5. For loops
    • What a for loop is
    • When to use a for loop
    • Using a for loop with range()
    • How to know which loop to use (for, while)
  6. Functions
    • What a function is
    • Understand that a function should do one thing
    • How to pass arguments into a function
    • What default arguments are
    • What a docstring is and what they're for
    • Usage of the return keyword to get things out of a function
    • Why we generally do not access variables outside of scope of a function
  7. [Extension] Where to next
    • Classes
    • Doctesting
    • Unit Testing
    • Version control (Git)
    • Using virtual environments to install non-standard libraries

Misc Concepts

  • Running code in your head
  • The DRY Principle
  • Comments
  • Problem Decomposition
  • Input validation
  • Loading data from a file
  • Saving data to a file
  • Usage of f-strings
  • Using random numbers

Themes of this approach

Topics are sequential except for Misc.

  1. New syntax should be introduced as a group and then used thoroughly.
  2. Questions should be deliberately using as much prior syntax as possible.
  3. Key ideas should be emphasised in each section.
